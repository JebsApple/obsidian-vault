## Diapositiva 3

Diagrama de actividades
El diagrama de actividades es un tipo de diagrama de UML que representa el flujo de actividades o pasos que se siguen para completar un proceso.
Este tipo de diagrama se usa en análisis y diseño de sistemas para modelar procesos: por ejemplo, cómo fluye un caso de uso dentro de una app, o cómo interactúan varios usuarios (clientes, empleados, sistema).

## Diapositiva 6

Diagrama de actividades
1- Registro de usuario en una aplicación web
Un usuario quiere crear una cuenta. Debe ingresar sus datos, validarlos, aceptar los términos y confirmar el registro a través de un correo electrónico.
2- Proceso de compra en una tienda online
El cliente navega por productos, selecciona uno, lo agrega al carrito, ingresa datos de pago, confirma la compra y recibe un correo de confirmación.
3- Ingreso de un alumno a clases virtuales
El estudiante entra al sistema, inicia sesión, verifica la disponibilidad de la clase, accede al aula virtual o espera si aún no está habilitada.
4- Solicitud de vacaciones en una empresa
El empleado solicita días de vacaciones. El jefe revisa la solicitud y la aprueba o la rechaza. Si se aprueba, Recursos Humanos actualiza el calendario laboral.
5- Proceso de devolución de producto
El cliente inicia una devolución, explica el motivo, el sistema verifica si está dentro del plazo. Si se aprueba, se emite una guía de envío.
6- Publicación de un artículo en un blog
El autor redacta un artículo, lo guarda como borrador, lo revisa un editor, y si está aprobado, se publica. Si no, vuelve a revisión.

## Diapositiva 7

Diagrama de clases
7- Proceso de inscripción a materias en una universidad
El alumno inicia sesión en el sistema, consulta las materias disponibles, selecciona las materias, verifica si hay cupo y confirma la inscripción.
8- Envío de un correo electrónico con archivo adjunto
El usuario redacta un mensaje, adjunta un archivo, el sistema valida el tamaño y el formato, y si todo está correcto, lo envía. Si hay error, muestra advertencia.
9- Pago de una factura de servicios (luz, agua, etc.)
El usuario ingresa al portal de pagos, selecciona el servicio, revisa la factura, elige método de pago, paga y recibe comprobante.
10- Evaluación de un examen online
El estudiante ingresa al sistema, realiza el examen dentro del tiempo, el sistema evalúa automáticamente las respuestas y muestra el resultado final.
11- Reserva de turno en un centro médico
El paciente inicia sesión, selecciona especialidad, elige médico y horario disponible, confirma el turno y recibe notificación por correo.
