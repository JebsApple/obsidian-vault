# TomaRequerimientosSol

Extraído de `TomaRequerimientosSol.docx`

---

Requerimientos Funcionales
Cliente
RF01: El sistema debe permitir al cliente seleccionar un horario de cancha.
RF02: El sistema debe bloquear temporalmente el horario seleccionado por el cliente mientras realiza el proceso de reserva.
RF03: El sistema debe confirmar la reserva y marcar el horario como ocupado una vez que el cliente complete el pago.
RF04: El sistema debe permitir al cliente realizar reservas y pagos desde la aplicación móvil.
RF05: El sistema debe permitir al cliente renovar su suscripción desde la plataforma.
RF06: El sistema debe informar al cliente cuando una clase se encuentre completa.
RF07: El sistema debe impedir que el cliente se inscriba en una clase sin cupos disponibles.
Recepcionista
RF08: El sistema debe permitir al recepcionista buscar clientes mediante RUT o nombre.
RF09: El sistema debe mostrar al recepcionista el estado del cliente mediante indicadores visuales (al día o con deuda).
RF10: El sistema debe permitir al recepcionista registrar el ingreso (check-in) de un cliente con una acción rápida.
RF11: El sistema debe mostrar al recepcionista un panel principal con información relevante para la atención.
RF12: El sistema debe notificar al recepcionista mediante alertas visuales cuando se registre una nueva reserva o suscripción.
RF13: El sistema debe emitir alertas sonoras en la interfaz utilizada por el recepcionista ante nuevos eventos.
RF14: El sistema debe actualizar automáticamente la información visible para el recepcionista sin necesidad de recargar la página.
Sistema (reglas automáticas)
RF15: El sistema debe evitar reservas simultáneas para un mismo bloque horario.
RF16: El sistema debe actualizar en tiempo real la disponibilidad de canchas.
RF17: El sistema debe gestionar un contador de cupos por cada sesión de clase.
RF18: El sistema debe bloquear nuevas inscripciones cuando se alcance el límite de cupos.
RF19: El sistema debe detectar cuando una suscripción esté vencida o impaga.
RF20: El sistema debe bloquear o generar alertas cuando un cliente tenga su suscripción vencida.
Administrador / Dueño
RF21: El sistema debe generar para el administrador un reporte de cierre de caja diario.
RF22: El sistema debe permitir al administrador visualizar ingresos desglosados por categoría de servicio.
RF23: El sistema debe permitir al administrador visualizar ingresos desglosados por método de pago.
RF24: El sistema debe permitir al administrador supervisar el estado del negocio mediante reportes en tiempo real.
Requerimientos No Funcionales
RNF01: El sistema debe reflejar cambios en reservas y pagos en tiempo real para clientes y personal de recepción.
RNF02: El sistema debe actualizar la información sin necesidad de recarga manual por parte del recepcionista.
RNF03: La interfaz utilizada por el recepcionista debe permitir ejecutar acciones críticas en el menor tiempo posible.
RNF04: El sistema debe ser accesible desde navegadores web en computadores de recepción.
RNF05: El sistema debe contar con una aplicación móvil para clientes y administrador compatible con iOS y Android.
RNF06: El sistema debe garantizar la consistencia de los datos evitando conflictos en reservas simultáneas.
