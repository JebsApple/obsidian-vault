## Diapositiva 6

Software de sistemas
Conjunto de programas escritos para dar servicio a otros programas.
Ciertos software de sistemas (por ejemplo, compiladores, editores y herramientas para administrar archivos) procesa estructuras de información complejas pero deterministas.
Otras aplicaciones de sistemas (por ejemplo, componentes de sistemas operativos, manejadores, software de redes, procesadores de telecomunicaciones) procesan sobre todo datos indeterminados.

## Diapositiva 7

Software de aplicación
Programas independientes que resuelven una necesidad específica de negocios. Las aplicaciones en esta área procesan datos comerciales o técnicos en una forma que facilita las operaciones de negocios o la toma de decisiones administrativas o técnicas.

## Diapositiva 8

Software científico/de ingeniería
Son un amplio conjunto de programas científicos de "cálculos numéricos". Las aplicaciones van desde la astronomía a la vulcanología, de la biología molecular a la manufactura automatizada.

## Diapositiva 9

Software integrado
Reside dentro de un producto o sistema y se usa para implementar y controlar características y funciones para el usuario final y para el sistema en sí. El software incrustado ejecuta funciones limitadas y particulares (por ejemplo, control del tablero de un horno de microondas).

## Diapositiva 10

Software de línea de productos
Consiste en componentes reutilizables y esta diseñado para proveer las capacidades especificas que puedan usar muchos clientes distintos. Puede enfocarse en un mercado limitado y particular (por ejemplo, control del inventario de productos) o se dirige a mercados masivos de consumidores.

## Diapositiva 11

Aplicaciones web/móviles
Esta categoría de software esta centrada en las redes cubre una amplia gama de aplicaciones y abarca las aplicaciones basadas en el navegador, la computación en la nube, la computación basada en servicios y el software que reside en los dispositivos móviles.

## Diapositiva 12

Software de inteligencia artificial
Utiliza la heurística para resolver problemas complejos que no son fáciles de tratar computacionalmente o con el análisis directo. Las aplicaciones en esta área incluyen robótica, sistemas expertos, reconocimiento de patrones (imagen y voz), redes neurales artificiales, demostración de teoremas y videojuegos.

## Diapositiva 13

Software Heredado
Es el software que fue desarrollado hace varias décadas y se han modificado de manera continua para cumplir con los cambios en los requerimientos de negocios y las plataformas informáticas. Por lo general su mantenimientos es costoso y su evolución es riesgosa. Puede generar un efecto secundario como la mala calidad.

## Diapositiva 15

Ingeniería de software
Es la aplicación de enfoques sistemáticos, disciplinados y cuantificables al desarrollo, operación y mantenimiento de software y el estudio de estos enfoques [1].
La ingeniería de software es una disciplina de ingeniería que se interesa por todos los aspectos de la producción de software, desde las primeras etapas de la especificación del sistema hasta el mantenimiento del sistema después de que se pone en operación [2].
[1] IEEE, Standard glossary of software engineering terminology, IEEE Std 610.12-1990, 1990.
[2] I. Sommerville, Software Engineering, 9th ed. Addison-Wesley, 2011.

## Diapositiva 17

Capa de Calidad del Software
Es la base fundamental de la Ingeniería de Software. Garantiza que el software cumpla con los atributos de calidad como confiabilidad, mantenibilidad y eficiencia. Se logra a través de prácticas como revisiones, pruebas y estándares de calidad.

## Diapositiva 18

Capa de Procesos
Define el conjunto de actividades necesarias para desarrollar software de manera efectiva.
Establece un marco de trabajo para la planificación, desarrollo y mantenimiento del software. Es la base del control administrativo de los proyectos de software.

## Diapositiva 19

Capa de Métodos
Comprende técnicas y prácticas utilizadas en el desarrollo del software.
Incluye metodologías para el análisis, diseño, implementación, prueba y mantenimiento del software.
Ejemplos: UML para modelado, patrones de diseño, principios SOLID, etc.

## Diapositiva 20

Capa de Herramientas
Proporciona el soporte automatizado para aplicar los métodos y procesos de software.
Incluye IDEs, herramientas de modelado, control de versiones, herramientas de prueba y CI/CD.
Ejemplos: Eclipse, Visual Studio, Git, Jenkins, SonarQube, etc.

## Diapositiva 21

Marco de trabajo del proceso
Comunicación:
La comunicación es crítica a la hora de colaborar con el cliente (y con otros participantes). Se busca entender los objetivos de los participantes respecto del proyecto, y reunir los  requerimientos que ayuden a definir las características y funciones del software.
Planeación:
Define el trabajo de ingeniería de software al describir las tareas técnicas por realizar, los riesgos probables, los recursos que se requieren, los productos del trabajo que se obtendrán y una programación de las actividades.
Modelado:
Habla sobre la creación de modelos a fin de entender mejor los requerimientos del software y el diseño que los satisfará.
Construcción:
Aquí se combina la generación de código y las pruebas requeridas para descubrir errores en el código.
Implementación:
Se entrega el software al cliente, quien evalúa el producto y ofrece retroalimentación.

## Diapositiva 22

Actividades sombrillas
Son actividades del marco de trabajo que complementan el desarrollo:
Seguimiento y control del proyecto de software
Ej. Trello para monitorear el avance de tareas
Gestión de riesgos
Ej. Identificación de riesgos en un proyecto ágil mediante reuniones de planificación.
Aseguramiento de la calidad del software
Ej. Implementación de SonarQube en el proceso de CI/CD para analizar la calidad del código
Revisiones técnicas
Ej. Realización de Code Reviews, donde otros desarrolladores revisan el código antes de fusionarlo en el repositorio principal.
Mediciones
Ej. Uso de métricas como Puntos de Función, Velocidad del Equipo en Scrum
Gestión de la configuración del software
Ej. Uso de Git para gestionar versiones del código y permitir la colaboración en equipo
Gestión de la reutilización
Ej. Creación de librerías y módulos reutilizables en proyectos de software para evitar reescribir código repetitivo
Preparación y producción del producto de trabajo
Ej. Crear documentación e informes de pruebas automatizados

## Diapositiva 23

Atributos de calidad del software
Mantenimiento:
El software debe escribirse de tal forma que pueda evolucionar para satisfacer las necesidades cambiantes de los clientes. Éste es un atributo crítico porque el cambio del software es un requerimiento inevitable de un entorno empresarial variable.
Confiabilidad y seguridad:
La confiabilidad del software incluye un rango de características que abarcan fiabilidad, seguridad y protección. El software confiable no tiene que causar daño físico ni económico, en caso de falla del sistema. Los usuarios malintencionados no deben tener posibilidad de acceder al sistema o dañarlo.
Eficiencia:
El software no tiene que desperdiciar los recursos del sistema, como la memoria y los ciclos del procesador. Por lo tanto, la eficiencia incluye capacidad de respuesta, tiempo de procesamiento, utilización de memoria, etcétera.
Aceptabilidad:
El software debe ser aceptable al tipo de usuarios para quienes se diseña. Esto significa que necesita ser comprensible, utilizable y compatible con otros sistemas que ellos usan.
