### Debug post-migración (21:20)

**Verificado:** `qs -c caelestia -n -d` corriendo OK (PID activo desde 20:41). Servicios viejos (waybar*, hyprsunset) confirmados disabled. dunst confirmado muerto, no respawnea.

**Bugs encontrados y arreglados:**
- `~/.local/share/applications/winrar.desktop` — `Exec=` con backslash sin escapar (`C:\Program Files\...`) rompía el parser de desktop entries de quickshell, generaba warning en cada refresh del launcher/systray. Fix: escapado a `\\`.
- `~/.config/mimeapps.list` — `inode/directory=dolphin.desktop` apuntaba a un ID de desktop file inexistente (el real es `org.kde.dolphin.desktop`). Por eso xdg-mime caía al cache del sistema y resolvía `kitty-open.desktop` como file manager por defecto (kitty también declara `inode/directory` en su MimeType). Fix: ID corregido, verificado con `xdg-mime query default inode/directory` → ahora devuelve dolphin correcto.

**Bugs sin arreglar (necesitan input tuyo o son upstream):**
- `.face` no existe en ningún lado del sistema — foto de perfil rota en lock screen (warning repetido en log). Necesita imagen real, no se puede fabricar.
- Terminal default en Nexus (GlobalConfig.general.apps.terminal) vive en estado runtime del shell, no en archivo estático — setear a mano en el panel Nexus (Settings → Apps → Terminal → kitty).
- `windowinfo/Preview.qml:93` — `TypeError: Cannot read property 'name' of null` al hacer hover sobre ciertas ventanas. Archivo pertenece al paquete `caelestia-shell` (`/etc/xdg/quickshell/caelestia/`), no se toca a mano porque un update de pacman lo pisa. Reportar upstream si molesta.
