# Modelos de Desarrollo de Software

## Waterfall (Cascada)
Modelo secuencial: cada fase debe completarse antes de la siguiente.

**Fases:** Requerimientos → Diseño → Implementación → Pruebas → Mantenimiento

**Problemas:**
- Cambios tardíos muy costosos
- Feedback hasta el final
- No adapta a cambios en requerimientos

## Modelo de Prototipos
Crear prototipos rápidos para descubrir requerimientos reales.

**Proceso:** Construir prototipo → Evaluar con usuario → Refinar → Repetir

**Problemas:**
- Puede convertirse en el producto final (deuda técnica)
- Los usuarios pueden confundir prototipo con producto terminado

## Modelo Espiral
Combina iteraciones con gestión de riesgos. Cada ciclo del espiral: Planificación → Evaluación de riesgos → Ingeniería → Evaluación del cliente.

**Ventajas:**
- Riesgos identificados temprano
- Iterativo (anticipa ágil)

**Problemas:**
- Requiere experiencia en gestión de riesgos
- Puede ser costoso para proyectos pequeños
