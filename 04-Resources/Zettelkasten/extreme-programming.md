# Extreme Programming (XP)

Metodología ágil que lleva las buenas prácticas al extremo (hacerlas continuamente).

## 4 Actividades Principales

### 1. Planning
- **Historias de usuario** como unidad de funcionalidad
- Planificación colaborativa cliente-desarrollador
- Estimaciones relativas (story points, no horas)

### 2. Design
- **CRC Cards** (Class-Responsibility-Collaboration)
- Simplicidad: diseñar solo lo necesario hoy
- Evitar complejidad anticipada (YAGNI)

### 3. Coding
- **Pair Programming:** dos personas, un workstation
- **Continuous Integration:** integrar código múltiples veces al día
- Estándares de código compartidos
- Code ownership colectivo

### 4. Testing
- **TDD (Test-Driven Development):** Red → Green → Refactor
- Pruebas de unidad escritas antes del código
- Pruebas de aceptación por el cliente
- Bugs detectados temprano = costo menor

## Prácticas Clave
- Refactoring continuo
- Metáfora del sistema (diseño simple)
- Coding standards
- 40-hour week (ritmo sostenible)
